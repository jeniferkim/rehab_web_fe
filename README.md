# 🏥 RehabAI Web – 홈 재활 코치 서비스

본 서비스는 가천대학교 컴퓨터공학과 P-실무프로젝트 젯디디(JCDD)팀에서 진행되었습니다. (2025.11.24 - 2025.12.18)

RehabAI는 의료 진단이 아닌 ‘운동 가능 범위 분류’를 중심으로
사용자가 **집에서도 안전하고 꾸준하게 재활을 이어갈 수 있도록 돕는 홈 재활 코치 서비스**입니다.

본 레포지토리는 RehabAI 서비스의 **웹 프론트엔드** 구현을 담당합니다.

---

## 📌 프로젝트 개요

* **프로젝트명**: RehabAI (Web)

* **목표**
  병원 방문 이전 단계의 사용자가
  자신의 현재 상태를 이해하고,
  무리 없는 재활 운동을 **스스로 시작·유지**할 수 있도록 지원

* **핵심 컨셉**

  * 의료적 *진단* ❌
  * 검증된 지표 기반 *운동 가능 범위 분류* ✅
  * 규칙 기반 판단 + AI 설명 중심 구조

---

## 🎯 주요 기능

### 1. 온보딩 & 자가 평가

* 통증 부위 / 통증 정도 / 기능 상태 입력
* VAS/NRS, 기능 평가 지표 기반 문진
* **Rule-based Triage**를 통한 상태 분류

  * 운동 가능
  * 주의 필요
  * 병원 방문 권고

### 2. 운동 루틴 제공

* 분류된 상태에 맞는 **안전한 운동 루틴**
* 운동 설명 + 영상 기반 가이드
* AI는 판단 주체가 아닌 **설명·코칭 역할**에 집중

### 3. 진행 현황 시각화

* 캘린더 기반 루틴 수행 기록
* 통증 변화 추이 차트
* 주간 하이라이트 및 회복 흐름 피드백

### 4. 사용자 경험(UX) 설계

* 스트릭(streak) 기반 지속성 강화
* 완료율 / 진행률 시각화
* “지금 잘 하고 있다”는 효능감 제공

---

## 🧠 AI 활용 방식

RehabAI의 AI는 **의료 판단을 하지 않습니다.**

* **Rule-based 로직**

  * 상태 분류 및 위험도 판단
  * 의료진 검증 가능 구조

* **LLM(GPT API)**

  * 사용자 상태 요약
  * 운동 설명 및 코칭 메시지 생성
  * 자연어 기반 이해 보조

> 판단은 규칙으로,
> 설명은 AI로 분리한 구조입니다.

## 📁 프로젝트 구조 (요약)

```
src/
 ├─ components/      # 공통 UI 컴포넌트
 ├─ pages/           # 페이지 단위 구성
 ├─ apis/            # API 호출 로직
 ├─ stores/          # Zustand 상태 관리
 ├─ types/           # 공통 타입 정의
```

## ⚠️ 주의 사항

* 본 서비스는 **의료 진단 및 처방을 제공하지 않습니다**
